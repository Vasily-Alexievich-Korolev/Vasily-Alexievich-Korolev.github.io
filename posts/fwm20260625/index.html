因子化世界模型：三次迭代、七次实验、全部亏损

发表于 2026-06-25 更新于 2026-06-26 分类于 Computer Science ， Machine Learning 阅读次数：
本文字数： 2k 阅读时长 ≈ 7 分钟
从Transformer趋势分解到VAE体制推断再到EMA监督编码器->收益预测，7次全量实验无一正收益，亏损率-77%~-100%。记录一次彻底但诚实的技术探索。

缘起

2025年底读到一篇关于因子化世界模型的论文，讲的是将图像分解为"语义token"和"细节token"进行分治建模。核心思路是：如果能把复杂系统的生成过程解耦成不同时间尺度的子过程，分别建模再组合，应该比端到端硬拟合更高效。

这个思路可以迁移到金融时间序列上。价格运动可以分解为两个基本物理过程：

  1. 趋势延续：价格沿着已形成的趋势惯性运行——低频、近似确定性的宏观过程。
  2. 均值回归：价格因短期偏离而出现向均值回归的倾向——高频、类似弹性恢复的过程。

于是开始了因子化世界模型（Factorized World Model, FWM）项目。前后迭代三个版本，七次全量实验，全部亏损，无一正收益。

V1：HP滤波 + Transformer（2026-06-25）

架构设计

从原始 OHLCV 中提取趋势状态序列和回归残差序列：

  1. HP滤波：\(\lambda = 5000\) 从收盘价提取平滑趋势线 \(\tau_t\)
  2. 趋势Token：计算趋势线滑动斜率的z-score，离散化为5类——强下降、弱下降、横盘、弱上升、强上升
  3. 残差特征：\(r_t = p_t - \tau_t\)，提取8维特征向量
  4. 均值回归标签：残差超过\(2\sigma\)且未来回归触发，标记为回归点

模型是4层Transformer Decoder（因果掩码，d=192, 6头），预测下一时刻的趋势Token分布和均值回归概率。

滚动训练结果

18个累积窗口的walk-forward测试，覆盖2022-2026全年：

第一个版本的失败是彻底的。原因是HP滤波的前视偏差——双向滤波使用了全部数据来估计某一点的趋势，训练集里已经包含了未来信息，到真没见过的新数据上自然崩溃。

延伸阅读：之前用DeepSeek-LLM做交易回测（50+技术指标，44%方向正确率）和XGBoost/随机森林的尝试，也都得到了同样的负结论。

V2：VAE + 三种概率体制（2026-06-26）

根本转变

V1的核心假设是"市场每一时刻都能被确定性地分解为趋势成分+回归成分"。但事实证明，这个假设太强了。市场大部分时间处于随机游走状态。

改为概率框架：市场存在三种不可观测的生成体制——趋势体制、回归体制、噪声体制。任何时刻市场仅处于其中一种体制，我们只能用概率推断：

\[P(\text{Trend}) + P(\text{Reversion}) + P(\text{Noise}) = 1\]

架构

采用VAE + Gumbel-Softmax离散潜变量框架。编码器是LSTM，输出3个类别的softmax概率。解码器由三个条件高斯生成器组成，分别对应三种体制，通过自相关系数约束（趋势正自相关、回归负自相关、噪声零自相关）让三类自动形成语义。

全量测试结果

指标 数值 含义
训练 17 epoch早停 快速收敛
体制分布 Trend 53%/Rev 44%/Noise 3% Trend/Rev几乎各半
Trend自相关 +0.936（目标+0.3） 约束失效
Reversion自相关 +0.996（目标-0.3） 完全错误方向
预测器 KL=12.9，训练不动 完全没有学习
回测 -77.91% vs BH +15.15% 大幅亏损

自相关约束被NLL损失压倒，三个生成器未能分化出不同动态模式。VAE实际学到的只是区分"高波动期"和"低波动期"，而非"趋势"和"回归"——这跟传统HMM的失败模式完全一样。

V3：EMA监督编码器（2026-06-26）

突破性的想法

前两次失败有一个共同的根因：模型分出来的状态（Trend/Rev/Noise），和我们人类理解的趋势/回归，根本不是一回事。自相关约束这种弱结构先验不足以让神经网络区分正自相关和负自相关动力学。

改用EMA20/EMA50规则直接写死状态定义：

  • UPTREND：EMA20 > EMA50，EMA20斜率>0，价格在EMA20之上
  • DOWNTREND：EMA20 < EMA50，EMA20斜率<0，价格在EMA20之下
  • NOISE：均线缠绕、收敛、或价格无序穿越

然后用LSTM从不含均线的原始特征中识别当前状态——让模型学会"看懂"EMA规则。

第一个真正的成功

编码器测试集准确率：86.22%（UP=88.9%, DOWN=90.0%, NOISE=82.3%）

这是三个版本中第一次模型真正学到了有语义的东西——它的内部隐藏状态确实编码了"什么是上涨趋势、什么是下跌趋势、什么是震荡"。而且它是从原始K线形态、波动率、成交量等底层特征中学到的——不是在计算EMA，而是在识别导致EMA交叉的价格行为模式。

但回测结果：-88.40%。

即使完美地知道当前市场处于EMA定义的什么状态，也无法用这个信息盈利。原因很简单：市场已经为公开的EMA信息定价了。当金叉确认时，趋势已经走完了一大段，剩下的空间风险回报比极差。

切换事件预测（方向A）

编码器的87%准确率是一个坚实的地基，但预测器面对的是一个残酷的数据分布：

指标 数值
切换比例 12.5%
"明天=今天"基线 87.35%
预测器准确率 86.6%
切换召回率 65.9%
切换精确率 30.28%

切换精确率30.28%揭示了根本问题：模型喊了太多次"狼来了"。70%的切换预测是假警报，每一次假警报都产生一笔亏损交易。

最终尝试：收益方向预测

放弃预测体制切换。冻结编码器参数，在它的隐藏向量上训练一个MLP，直接预测下一期收益方向。

结果：验证准确率52.5% vs 基线50.8%。测试集51.70% vs 基线50.45%。

这个数字是三个版本七次实验中"最接近成功"的一次——但它仍然不足以覆盖交易成本。

阈值 收益 交易
0.55 -98.82% 4279
0.60 -18.72% 138
0.70 接近0% < 50

阈值抬到0.70后几乎不交易了，说明预测器输出集中在50-55%之间，从未达到过可以稳定交易的置信度。

三个版本的对比总结

维度 V1: HP滤波+Token V2: VAE+体制 V3: EMA监督
核心方法 人工趋势分解 无监督体制学习 监督标签学习
编码器质量 数据泄露 模式坍缩 87%准确✅
预测器 随机水平 完全失败 52.5%
最佳回测 -99.9% -77.91% -18.72%(th=0.60)
核心教训 前视偏差必死 弱先验不足以区分动力学 EMA知识无Alpha

终极结论

七次全量实验（V1×18窗口 + V1山寨币×3 + V2 + V3×3模式 + V3收益预测），覆盖了从人工规则分解、无监督VAE推断、到EMA监督学习、直接收益预测的全部路径。结果一致：

BTC 1h的纯OHLCV价格预测任务，信噪比极低，接近随机游走。 无论使用何种架构，都无法从中提取出能稳定覆盖交易成本的预测信号。

这跟之前一系列独立实验的结论完全一致： - DeepSeek-LLM交易回测：44%方向正确率（接近随机） - XGBoost/随机森林：无法稳定盈利 - LSTM集成：同样负预期

这个方向的投入产出比极低。 之前验证有效的策略（低波山寨币暴跌抄底，胜率57.5%，夏普4.01）继续实盘模拟。
